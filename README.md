## What SABLE Is

SABLE is not a trading bot. It is not a signal feed. It is not a dashboard with charts.

SABLE is an autonomous capital intelligence system — a governed, multi-agent deliberation architecture that ingests real-time market sensor data, fuses it across a structured ontology, debates it through an eight-agent adversarial council, enforces it against a constitutional law layer, and delivers sourced, annotated, fully structured investment cases to a single operator.

Where most systems chase signals, SABLE builds conviction.

It operates with the posture of a senior portfolio manager and the precision of a weapons system. **Silent until certain. Lethal when it acts.**

## The Shadow Council

Every thesis that reaches conviction has survived eight independent analytical lenses — a formal deliberation process where each agent produces a structured verdict object, not a sentiment, not a score, not a string.

| Agent | Mandate | Kill Authority |
|---|---|---|
| **ORACLE** | Regime classification. Runs first. Conditions all downstream logic. | — |
| **VECTOR** | Institutional flow intelligence. Translates sensor data into positioning theses. | — |
| **AXIOM** | Statistical edge. Win rate, expectancy, R-multiple viability. No edge, no trade. | — |
| **REFLEX** | Second-order effects. Soros-style feedback loop detection and exhaustion signals. | — |
| **SPECTER** | Adversarial intelligence. Sole mandate: find the reason this trade is wrong. | 1.5× weight |
| **PHANTOM** | Tail risk. VIX structure, credit spreads, cross-asset systemic scenarios. | — |
| **WARDEN** | Constitutional officer. Enforces the Five Laws. Issues REDLINEs. | **Absolute** |
| **ARBITER** | Synthesis. Does not vote. Produces the conviction score and Decision Package. | — |

**SPECTER + PHANTOM consensus** triggers a mandatory second deliberation — the first round is discarded, all agents re-vote with the adversarial case explicitly surfaced. Two adversarial agents in agreement is a hard stop.

**WARDEN** holds absolute kill authority. A REDLINE terminates deliberation immediately. No appeal. No override. The constitutional check is structural — not advisory.

```

        a Decision Package, operator authorization, and SABLE routing
```

A morning intelligence brief. Not a signal alert. Situational awareness: current regime classification, active theses in deliberation, book exposure, overnight risk events, macro calendar, PHANTOM risk level, and — when earned — a system self-portrait of current operating edges and known blind spots.

## Design Principles

**Ontology-First** — Raw sensor data is input. Ontology state is output. No raw data reaches the operator. Every signal passes through the ontology engine before it becomes a claim.

**Conviction-Gated** — Nothing reaches the operator without a conviction score earned through the full deliberation process. 3–5 Decision Packages per trading day is the ceiling, not the target.

**Provenance-Complete** — Every claim traces to an evidence source. Every source traces to an ingestion event. A package with an unsourced claim is rejected before delivery.

**Regime-Conditioned** — All conviction thresholds, position sizing, and council vote weights are conditioned on the current regime classification. A component that ignores regime is operating without context.

**Constitutional-First** — WARDEN evaluates every thesis before deliberation begins and again before the Decision Package is assembled. The constitutional check is not a final gate. It is a continuous constraint.

**Silence is Valid** — SABLE generating no Decision Package on a given day is correct behavior when no thesis earns conviction. Silence is not a bug. It is epistemological discipline.

## Operating Posture

> *SABLE is built to be wrong gracefully and right precisely.*
>
> *When SABLE is wrong — and it will be wrong — the error must be traceable.*
> *Which sensor fired a false signal. Which Council agent failed to catch it.*
> *Which constitutional check should have blocked it.*
>
> *When SABLE is right, the conviction was earned through process.*
> *The process is the product. Not the outcome. Not the P&L.*
> *P&L is the validation of the process — lagging, noisy, and subject to*
> *market randomness. The process is the thing under engineering control.*
>
> *Build the process. Let the P&L follow.*
